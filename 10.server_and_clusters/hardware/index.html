    硬件篇¶

    简介

    本节讲解对服务器物理机进行运维所需的知识，包括：

    • 服务器的硬件组成
    • 服务器的启动方式
      • BIOS、UEFI、PXE 等概念
    • 救援方法：服务器无法启动怎么办？

    服务器的硬件组成¶

    我们从我们熟悉的个人电脑开始，看看它的硬件组成，并与服务器对比。

    个人电脑

    这是我们最常见的个人电脑，一般是立式的。个人电脑一般只有一个 CPU 和较少的内存插槽。

    服务器

    这是一台典型的 2U 服务器（U 是服务器机体高度的单位）。服务器常常具有双路 CPU 和大量的内存插槽、存储单元。这里是图中展示的 Intel R1000WF 服务器的说明书，你可以在里面看到这台服务器的更多细节。除了图中的机架式，服务器还有刀片式、塔式等形态。

    机柜

    多个单体服务器放置在机柜中，与相关配套设施如 UPS（Uninterrupted Power Supply，不间断电源）、防火墙等组成一套完善的服务器系统。

    数据中心

    数据中心的主要目的是运行应用来处理商业和运作的组织的数据。除了数据库、文件服务器、应用服务器等构件，还包含冗余和备用电源供应，冗余数据通信连接，环境控制（例如空调、灭火器）和各种安全设备

    HPC 集群

    一个 HPC 通常有节点、网络、存储等部分。节点分为入口和管理节点、CPU 节点、GPU 节点等。节点之间通常依靠高速网络互联，组成网络存储系统和并行计算系统。

    以下是服务器和集群运维经常使用的硬件设施：

    PDU 电源分配单位

    Power Distribution Unit，是为机柜式安装的电气设备提供电力分配而设计的产品。PDU 通常具有独立的网络接口，能够接入网络设施进行远程控制。当服务器需要物理断电重启时，通常使用 PDU 进行操作。

    InfiniBand

    “无限带宽”技术，是超级计算机最常用的网络通信标准，具有极高吞吐量和极低延迟。InfiniBand 也提供远程直接内存访问（RDMA）能力以降低 CPU 负载。图示的 NVIDIA InfiniBand 将 DPU 集成在网络适配器上，这能够节省更多 CPU 计算资源，是数据中心经常采取的做法。

    系统的启动过程¶

    • BIOS 使用磁盘起始处的一些扇区来存储启动程序，而 UEFI 使用磁盘上的一个特殊分区来存储启动程序。UEFI 具有更多的功能，支持更大的磁盘空间等，正在逐渐取代 BIOS。
    • GRUB（GRand Unified Bootloader）是一个多重启动管理器，用于在多个操作系统共存时选择引导哪个操作系统。目前主流的 GRUB 版本是 GRUB2，它的配置文件是 /etc/default/grub。一个小技巧，开机时按住 Shift 键可以进入 GRUB 的启动菜单。
    UEFI 启动太快了，根本来不及按键进入 GRUB 菜单！

    可以修改 /etc/default/grub 文件，其中 GRUB_TIMEOUT_STYLE 的值很可能是 hidden。将其改为 menu，然后执行 sudo update-grub 命令，这样就可以在开机时进入 GRUB 菜单了。

    系统启动的各种方式¶

    在使用个人电脑时我们基本不会见到其他的启动方式，而在服务器运维时你可能见到 PXE、USB、CD、iPXE、ISO 等启动方式。

    • PXE（Preboot eXecution Environment）：预启动执行环境，是一种远程启动技术，可以通过网络从远程服务器启动计算机。PXE 服务器通过 DHCP 协议为客户端分配 IP 地址，然后通过 TFTP 协议为客户端提供启动文件。PXE 服务器可以是独立的服务器，也可以是路由器、交换机等网络设备。
    • iPXE：是 PXE 的改进版，支持更多的网络协议，如 HTTP、iSCSI 等。这些网络协议比 TFTP 更快也更可靠。
    • ISO 启动方式与上面几种启动方式的层级不一样：上面几种启动方式都是在硬件层面，由 BIOS/UEFI 选择，而 ISO 是在进入引导程序如 GRUB 后选择。
